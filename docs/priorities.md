# Habesha Market — Feature Priorities (MoSCoW Method)

This document ranks features by importance using the **MoSCoW** method:
- 🔴 **Must Have** — Project fails without these
- 🟡 **Should Have** — Important but not critical
- 🟢 **Could Have** — Nice to have if time permits
- ⚪ **Won't Have** — Not in this version

---

## 🔴 Must Have (Build First!)

- User registration & login (JWT auth)
- Role-based access (customer, seller, admin)
- Product listing on homepage
- Product details page
- Add to cart functionality
- Checkout & place order
- View order history
- Seller can add/edit/delete products
- Admin can view all users and products

---

## 🟡 Should Have (Build If Time Allows)

- Search products by name
- Filter products by category
- Filter products by price range
- Update cart item quantities
- Order status tracking (pending → shipped → delivered)
- Seller dashboard with own product list
- Admin dashboard with basic statistics
- Form validation on all inputs
- Responsive design (mobile-friendly)

---

## 🟢 Could Have (Bonus Features)

- Product image upload
- Multiple product categories
- Sort products by price/newest
- Pagination on product list
- Order confirmation page
- Basic loading spinners
- Toast notifications for actions

---

## ⚪ Won't Have (Not in v1.0)

- Online payment gateway (Stripe, Chapa, Telebirr)
- Product reviews and ratings
- Wishlist feature
- Real-time chat between buyer & seller
- Email/SMS notifications
- Password reset via email
- Profile pictures and user avatars
- Multi-language support (Amharic, Tigrinya)
- Mobile native app
- AI product recommendations

---
